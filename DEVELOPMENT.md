# Draw Tarot 開発メモ

## 目的

GitHub Pagesなどで静的ホストできる、タロットカードを引くWebアプリ。

## 現状

- 構成は `index.html`、`style.css`、`script.js`、`images/`、`test/static.test.js`。
- カード番号、名称、解説本文は `card-meanings.js` の `window.cardMeanings` で管理する。
- 大アルカナ22枚と裏面画像は `images/` に配置済み。
- 画像はファイル側で縦向きに正規化済み。
- カード画像はRWS系の新ファイル名を使う。画像差し替え時は `script.js` の `imageFiles` とテストを更新する。
- カットは「カードを任意の位置で上下に入れ替える」動作。
- スマホでは1画面に収め、引いたカードをできるだけ大きく表示する方針。
- スマホ初期画面はデッキと操作を優先し、結果カードと解説は操作の下に続けて表示する。
- ドロー後はカードが下へ抜ける演出を挟み、結果カードへ自動スクロールする。
- 結果にスクロールした後、引いたカードはユーザへ渡したものとして扱い、残りカードは揃えて一山のデッキに戻す。

## 体験設計

- このアプリは占い結果を断定するものではなく、問いを見る角度を変えるためのワンカードオラクルとして作る。
- 初期画面では、デッキ、問いの注意書き、操作ボタンを主役にする。
- 「大アルカナ22枚」のような説明は初期画面から外し、操作に必要な情報だけを残す。
- 注意書きはデッキ上に半透明で重ねる。背景は `rgba(21, 18, 15, .62)`、軽い `backdrop-filter: blur(3px)` を使う。
- 注意書きには「問いの例」を明示し、3つ全部を心に置くように読めない文言にする。
- シャッフル開始後、少し待ってから注意書きをゆっくりフェードアウトする。
- スクロールは標準の `scrollIntoView` ではなく、減速して止まる `easeOutCubic` の自前スクロールにする。

## レイアウト方針

- カード枠は画像比率に合わせて `--card-ratio: 3 / 5` を使う。
- カード画像は `object-fit: contain` で全体を見せる。絵柄を切らない。
- スマホではボタンが初期画面に確実に見えることを優先する。
- 結果カードは大きめに表示し、その下にカード解説を続ける。
- 結果表示後だけ縦スクロールを許可し、ユーザが解説まで読めるようにする。

## データ管理

- 解説本文は `card-meanings.js` だけを編集すれば差し替えられる形にする。
- 各カードは `number`、`name`、`imageDescription`、`positiveKeywords`、`negativeKeywords`、`treeOfLife` を持つ。
- 解説は「占い結果」ではなく、カードの象徴、絵柄、キーワード、生命の樹上の位置づけとして書く。

## テスト

```sh
npm test
```

確認内容:

- 画像23枚の存在
- 画像が縦長
- 画像orientationが正規化済み
- HTMLのCSS/JS参照
- CSSの主要レイアウト指定
- カード解説データ22枚分の存在と項目
- JS構文
- 注意書き、結果配置、スクロール、デッキ整列など主要UX指定

## 注意

- `npm test` 時にNode v16由来のnpm警告が出ることがあるが、現状テストは通る。
- 画像処理にはローカルの ImageMagick `magick` を使っている。
- GitHub Pagesで動かす静的HTMLなので、開発時テスト以外のビルド工程は増やさない。

## コミット方針

- コミット前に必ず `git diff HEAD` を確認する。
- diffから変更内容を要約してコミットメッセージを決める。
- 可能ならコミット前に `npm test` を実行する。
- push前に `git status --short` で意図しない変更がないか確認する。
- ユーザが編集した未コミット変更がある場合は、依頼対象だけを明示的にstageする。
